Vanzeiro

Previous Next

  cabuum/vanzeiro   vanzeiro.herokuapp.com


O vanzeiro é um projeto nobre que visa resolver os problemas enfrentados por donos de pequenos negócios, envolvendo transporte escolar (privado). Ele é uma espécie de ERP voltado à transporte escolar, desde o controle de frotas, rotas, financeiro, passageitos, viagens, até plataforma de venda de mini-vans e micro ônibus estilo OLX e aplicativo de controle de entrada/saída e rastreio de vans.

Na Cabuum trabalhei juntamente com o @joridos no desenvolvimento do vanzeiro, é sem dúvida o 1° grande projeto escrito em rails pela gente.

O projeto inicialmente foi desenvolvido usando MySQL e AWS como servidor, usávamos puma para server app. Agora está usando Heroku e PostgresSQL. Na época não entendia os benefícios dos testes unitários, e portanto não o foram realizados.

Atualmente o projeto está em fase de reestruturação, aguardando novas iterações.

  • Cliente: Cabuum
  • Data: Setembro 2014
  • Categorias: Web Startup Ruby on Rails AWS

Poolíticos

Previous Next

  brunoocasali/pooliticos   pooliticos.com


O poolíticos é feito com base no tema "Creative" do StartBootstrap. Ele foi projetado e desenvolvido durante uma sprint de 2 semanas.

Ele pretende coletar o interesse dos candidatos com o seu eleitorado, a ideia é que com base de dados coletados automáticamente e/ou dados comprados seja possível agregá-los e enviar uma propaganda mais focada nos interesses do lead. Lead esse que está conectado à internet e por tanto muito mais acessível do que ao utilizar os métodos antiquados de marketing (santinhos, carro de som e folders). Economizando muito tempo e aumentando a taxa de conversão.

O web site conta com um sistema extremamente simples, que dá total controle do grupo de usuários do partido, ou político para enviar cadastrar os leads e enviar os conteúdos.

  • Cliente: Alan Darold
  • Data: Julho 2016
  • Categorias: Web Startup Ruby on Rails Postgres Heroku

Interactive Event (telão)

Previous Next

  brunoocasali/interactive-event   telao.herokuapp.com


O telão é uma criação que usa Material Design. Projeto muito bacana que visava acompanhar as redes sociais sobre uma determinada hashtag, e então aprovar ou não tais conteúdos.

A sacada do projeto é exatamente a de mostrar ou não X tweets, o facebook dropou o suporte da api que procurava posts dentro do endpoint de hashtags, por isso não pude fazer o crawler no facebook apenas instagram e twitter. Outro grande benefício são as tecnologias usadas, apenas tecnologias free estão sendo usadas, como o Dropbox, usei a conta de user normal, para armazenar as fotos, e lidei com a latência entre uma requisição de lista de fotos carregando uma a uma alterado o design da lista, funcionou perfeitamente!

O projeto foi usado durante o evento da Miss Francisco Beltrão 2015, e foi um sucesso!

  • Cliente: RG2 Produções
  • Data: Março-Agosto 2015
  • Categorias: Web Freelancer Ruby on Rails Postgres Heroku

Zueira F.C.

Previous Next

  brunoocasali/zueirafc   zueirafc.com


A minha mais completa e complexa criação (por enquanto)! O Zueira Futebol Clube, busca reunir por meio de uma lista de criadores de imagens de humor (futebolístico) escolhida a dedo e agregar isso tudo em um só lugar.

A cada X tempo o crawler (rails api) itera sobre a lista de fontes busca de diversas fontes diferentes várias mídias de "zueiras", seja texto, imagem, vídeo tudo. Agrega e manipula de acordo com as regras pré-estabelecidas. Então as mídias passam por uma fase de triagem para remover as impurezas (conteúdo sem relacionamento com o tema), para só então serem exibidas na página inicial.

O projeto foi desenvolvido com Ruby on Rails, Postgres, Heroku, Node.js, angular.js e cron jobs, além de usar o Phusion Passenger como server de aplicação. Além de ter uma ótima cobertura de testes unitários e integração contínua tanto no wep app quanto na api.

  • Data: Dez 2015 - Junho 2016
  • Categorias: Web Startup Ruby on Rails Node.js Postgres Heroku
